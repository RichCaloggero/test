To make these kind of tables accessible::

  • provide both row and column header cells with ID attributes.
  • Add aria-labelledby to each radio button referencing the appropriate column header's ID.
  • Be sure each radio button has an name attribute; this connects each button in each row together and allows arrow keys to select each in turn.
  • Finally, the first radio button in each row should have the ID of the row header cell as the first ID reference in it's aria-labelledby value.

Noe: please do not link all radio buttons in each row to the row header via aria-labelledby; this will cause the screen reader to repeat the question (contents of row header cell) on each tab press, which would be extremely verbose and make the widget virtually unusable!
